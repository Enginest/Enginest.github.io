Troubleshooting

Server Connection​

AWS​

Attention

Si votre serveur est sur AWS avec une base de donnée Access et que vous venez de démarrer l'instance il est possible que vous obtenez l'erreur "failure: constraint ' [databaseName:databaseUser:databasePassword] ' (CONNECTDB) at rotation 0 at ' [databaseName:databaseUser:databasePassword] ' (CONNECTDB)", dans ce cas merci de suivre les étapes ci-dessous.

  • Se connecter sur l'instance EC2 sur AWS en RDP
  • Aller dans l'application "Access"
  • Dans le menu compte (Echap > Compte) mettez à jour l'application
    Cela devrais régler le problème.

Local​

Si vous n'arrivez pas a vous connectez au serveur en local. Verifiez ici.

Bad Configuration​

Danger

Assurez-vous d'avoir bien configuré le fichier "mobile.config" sinon il risque d'avoir de la casse.
Configuration ici.

Comment vérifier ?​

Note

Vous pouvez modifier le comportement du parsing dans le fichier "EditOrder.js" ou/et "EditOrderData.js"

Astuce

Si un jour vous devez modifier le comportement du parsing, je vous conseille de le refaire de 0.
Cela prendra du temps mais c'est nécessaire car actuellement c'est une grosse "usine à gaz".

Relire la configuration.

  • Vérifiez que le nom de table est valide et en majuscule.
  • Vérifiez que les nombres correspondent bien (nombre de champs/nombre d'alias).
  • Vérifiez que les champs soient orthographiés de la même façon que dans la base de données.
  • Vérifiez que les champs soit dans l'ordre d'apparition dans la base de données.

Exemple :​

Note

J'explique 2 cas (SQL Script et Types/Alias), vous n'aurez aucun problème pour comprendre les autres partie de la configuration.

  • SQL Script
  • Types/Alias

Version dans la config :

TMS32/Config/00/mobile.config
OPT_FEEDBACK	4
idClub		"SELECT idClub, NULL FROM OPT_FEEDBACK"
idResource	"SELECT idResource, NULL  FROM OPT_TOUR"
tourResource	"SELECT tourResource, NULL  FROM OPT_TOUR"
KM		"SELECT KM, null FROM OPT_FEEDBACK WHERE KM = (SELECT MAX(KM) FROM OPT_FEEDBACK)"

Version du retour serveur :

OPT_FEEDBACK  // Nom de table
4 // Nombre de champs
idClub // Nom champ 1
"SELECT idClub, NULL FROM OPT_FEEDBACK" // Requête champ 1
idResource // Nom champ 2
"SELECT idResource, NULL  FROM OPT_TOUR" // Requête champ 2
tourResource // Nom champ 3
"SELECT tourResource, NULL  FROM OPT_TOUR" // Requête champ 3
KM // Nom champ 4
"SELECT KM, null FROM OPT_FEEDBACK WHERE KM = (SELECT MAX(KM) FROM OPT_FEEDBACK)" // Requête champ 4

Version dans la config :

TMS32/Config/00/mobile.config
OPT_FEEDBACK	4
t0		datetime	true
latitude	latitude	true
longitude	longitude	true

	0 	"position"
