Composite Key

DEFINITION
Une clé composite est constituée par la combinaison de deux colonnes ou plus dans un tableau qui peut être utilisée pour identifier de manière unique chaque ligne du tableau lorsque les colonnes sont combinées. L'unicité d'une ligne est garantie, mais lorsqu'elle est prise individuellement, elle ne garantit pas l'unicité, ou elle peut également être comprise comme une clé primaire créée par la combinaison de deux ou plusieurs attributs pour identifier de manière unique chaque ligne d'un tableau.

Vous pouvez modifier.

4
OPT_ORDER	idClient+id
OPT_RESOURCE	idClub+id
OPT_FEEDBACK	idClub+idResource+tourResource+ord
OPT_SITE	idClub+id

Comment ca marche ?​

  • Le "4" signifie le nombre de table et le nom des tables (souligné en rouge)
  • Pour chaque table il y a le nom des champs séparé par un '+' qui vont composés la clé composite (souligné en vert)

Attention

En cas de modification, il faut impérativement faire correspondre les nombres.
